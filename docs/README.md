## Prinsip Utama

- Dokumentasi ditulis sebelum implementasi.
- Semua dokumen menggunakan Bahasa Indonesia.
- Setiap keputusan penting harus dapat ditelusuri.
- Struktur ini tidak berisi kode aplikasi.
- Struktur ini tidak membuat database, UI, atau komponen.

## Aturan Bisnis

- Dokumentasi harus diperbarui saat keputusan produk berubah.
- Developer harus membaca dokumen terkait sebelum coding.
- Fitur baru harus memiliki referensi ke PRD dan feature catalog.
- Perubahan besar harus dicatat di changelog.

## Catatan Teknis

- Repository ini saat ini hanya berisi dokumentasi foundation.
- Implementasi aplikasi akan dibuat pada fase terpisah setelah dokumen disetujui.
