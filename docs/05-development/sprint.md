# Sprint Planning

## Tujuan

Dokumen ini menjadi template perencanaan sprint untuk pengembangan Vinari.

## Latar Belakang

Sprint planning membantu tim menerjemahkan roadmap menjadi pekerjaan yang jelas, terukur, dan dapat dievaluasi.

## Definition of Ready

- Requirement sudah jelas.
- Dependensi diketahui.
- Risiko utama dicatat.
- Acceptance criteria tersedia.

## Definition of Done

- Implementasi selesai.
- Review selesai.
- Test relevan selesai.
- Dokumentasi diperbarui.
- Tidak ada blocker kritis.

## Aturan Bisnis

- Pekerjaan tidak boleh masuk sprint tanpa tujuan yang jelas.
- Perubahan scope harus dicatat.
- Fitur sensitif harus memiliki review keamanan.
- Dokumentasi harus diperbarui jika keputusan berubah.

## Catatan Teknis

- Sprint belum dimulai pada tahap foundation.
- Dokumen ini hanya template untuk fase development.

## Pengembangan Masa Depan

- Menambahkan sprint history.
- Menambahkan velocity tracking.
- Menambahkan retrospective format.
