# Changelog

## Tujuan

Dokumen ini mencatat perubahan penting pada dokumentasi, keputusan produk, arsitektur, dan pengembangan Vinari.

## Latar Belakang

Sebagai single source of truth, dokumentasi Vinari harus memiliki riwayat perubahan agar keputusan dapat ditelusuri.

## Format Changelog

Gunakan format berikut untuk setiap perubahan:

```text
## [Tanggal] - Judul Perubahan

### Ditambahkan
- Item yang ditambahkan.

### Diubah
- Item yang diubah.

### Dihapus
- Item yang dihapus.

### Catatan
- Konteks atau alasan perubahan.
```
