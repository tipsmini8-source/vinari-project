# Design System

## Tujuan

Dokumen ini mendefinisikan arah awal design system Vinari tanpa membuat UI atau komponen.

## Latar Belakang

Sebagai produk finansial, Vinari harus terasa profesional, tenang, jelas, dan dapat dipercaya. Design system membantu menjaga konsistensi saat aplikasi berkembang.

## Prinsip Desain

- Jelas.
- Tenang.
- Profesional.
- Tidak menghakimi.
- Mudah dipindai.
- Mobile-first.
- Aksesibel.

## Elemen Design System

- Warna.
- Tipografi.
- Spacing.
- Radius.
- Elevation.
- Icon.
- Form.
- Table.
- Chart.
- Feedback state.
- Empty state.

## Arah Visual

Vinari harus menghindari tampilan yang terlalu ramai. Produk harus terasa seperti alat kerja finansial yang serius namun tetap ramah untuk pengguna personal dan keluarga.

## Aturan Bisnis

- Warna tidak boleh membuat status finansial terasa menghakimi.
- Error state harus jelas dan membantu.
- Data finansial harus mudah dibaca.
- Chart harus mendukung pemahaman, bukan dekorasi.

## Catatan Teknis

- Implementasi UI akan menggunakan Tailwind CSS dan shadcn/ui pada fase development.
- Dokumen ini belum membuat token desain final.
- Komponen belum dibuat pada tahap ini.

## Pengembangan Masa Depan

- Menentukan color palette.
- Menentukan typography scale.
- Membuat component guideline.
- Membuat accessibility checklist.
- Membuat data visualization guideline.
