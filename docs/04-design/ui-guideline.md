# UI Guideline

## Tujuan

Dokumen ini mendefinisikan pedoman UI awal untuk Vinari agar pengalaman pengguna konsisten, jelas, dan profesional.

## Latar Belakang

Vinari akan menampilkan data sensitif dan kompleks. UI harus membantu pengguna memahami data tanpa merasa kewalahan.

## Prinsip UI

- Informasi penting harus terlihat jelas.
- Aksi utama harus mudah ditemukan.
- Form harus ringkas dan terstruktur.
- Status keuangan harus ditampilkan secara empatik.
- UI harus mendukung penggunaan mobile.

## Pedoman Form

- Gunakan label yang jelas.
- Validasi harus muncul dekat field terkait.
- Field nominal harus mendukung format mata uang.
- Field tanggal harus mudah dipilih.
- Pengguna harus dapat membatalkan atau memperbaiki input.

## Pedoman Data Finansial

- Nominal harus konsisten formatnya.
- Warna status harus digunakan dengan hati-hati.
- Ringkasan harus menjelaskan konteks periode.
- Data kosong harus memiliki instruksi yang membantu.

## Pedoman Navigasi

- Workspace aktif harus terlihat.
- Area utama harus mudah dipahami.
- Fitur premium harus ditandai tanpa mengganggu alur free user.
- Mobile navigation harus sederhana.

## Aturan Bisnis

- UI tidak boleh menampilkan data yang tidak berhak diakses pengguna.
- UI tidak boleh membuat klaim finansial yang tidak didukung data.
- Upgrade prompt tidak boleh menghalangi fungsi inti free user secara agresif.

## Catatan Teknis

- Pedoman ini akan diterjemahkan ke komponen shadcn/ui pada fase development.
- Belum ada desain visual final pada tahap ini.
